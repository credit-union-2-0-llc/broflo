# Plan 08-03: Frontend Order Tracking

**Phase:** 08 — Order Tracking
**Depends on:** 08-02
**Estimated tasks:** 8

---

## Goal

Build /orders list page, /orders/:id detail page with status timeline, tracking info card, "Orders In Flight" dashboard widget, and navigation update.

---

## Tasks

### T-1: API client order methods

**File:** `apps/web/src/lib/api.ts` (or wherever the API client lives)

Add/update:
- `getOrders(params?: { status?, sortBy?, sortOrder?, page?, limit? })` → GET /orders
- `getOrder(id: string)` → GET /orders/:id
- `getOrderTimeline(id: string)` → GET /orders/:id/timeline

### T-2: Orders list page

**File:** `apps/web/src/app/(app)/orders/page.tsx`

- Status filter tabs: All, Active (ordered+processing+shipped), Delivered, Cancelled
- Sort dropdown: Newest first, Oldest first
- Paginated order cards showing:
  - Product title + image
  - Recipient name
  - OrderStatusBadge (reuse from S-7, add `delivered` variant)
  - Price
  - Date placed
  - Cancel countdown (if active + within window)
- Empty state: Broflo voice "No orders yet. Find someone a gift and we'll handle the rest."
- Loading skeleton

### T-3: Order detail page

**File:** `apps/web/src/app/(app)/orders/[id]/page.tsx`

Layout:
- Header: product title, status badge, price
- Status timeline (vertical, left-aligned):
  - Each entry: status icon + label + timestamp + source badge
  - Current status highlighted
  - Future states shown as grey/pending
- Tracking card (if shipped/delivered):
  - Carrier name, tracking number (copyable)
  - "Track Package" link → external carrier URL (target="_blank", rel="noopener")
- Product info card: image, title, description
- Recipient card: person name, shipping address
- Cancel button (if within window) with countdown + confirmation dialog
- Back link to /orders

### T-4: Status timeline component

**File:** `apps/web/src/components/orders/StatusTimeline.tsx`

Props: `entries: OrderStatusHistoryEntry[]`, `currentStatus: OrderStatus`

Vertical timeline with:
- Filled circle for completed statuses (green)
- Current status circle (blue, pulsing)
- Empty circle for future statuses (grey)
- Timestamp for each completed entry
- Source badge (system/webhook/manual) as subtle pill

### T-5: Tracking info card component

**File:** `apps/web/src/components/orders/TrackingCard.tsx`

Props: `trackingNumber?: string`, `trackingUrl?: string`, `carrierName?: string`

- Shows only when trackingNumber is present
- Copy button for tracking number
- External link to tracking URL (validated as https)
- Carrier name label

### T-6: Dashboard "Orders In Flight" widget

**File:** `apps/web/src/components/dashboard/OrdersInFlightWidget.tsx`

- Count of active orders (ordered + processing + shipped)
- Next expected delivery date
- Mini list of up to 3 active orders with status icon + product title
- "View all orders →" link
- Empty state: "All clear — no orders in transit"

Wire into dashboard page alongside existing widgets.

### T-7: Navigation update

**File:** `apps/web/src/app/(app)/layout.tsx` (or nav component)

Add "Orders" as 4th nav item between Events and Billing:
- Icon: Package or ShoppingBag from lucide-react
- Path: /orders
- Active state highlight

### T-8: Voice copy additions

**File:** `packages/shared/src/copy/voice.ts`

Add order tracking copy:
