    }));
  }
}

`async place(user: User, dto: PlaceOrderDto)`:
- Validate user has a payment method: `if (!user.stripePaymentMethodId) throw new BadRequestException('No payment method on file. Add one in Billing settings.')`.
- Validate suggestion ownership: same as preview.
- Get the product from adapter: `adapter.getProduct(dto.retailerProductId)`.
- Calculate platform fee: `Math.round(product.priceCents * 0.05)` (5% per confirmed decision).
- Create the Order record in DB with status `'pending'`, `placedAt: new Date()`, all shipping fields from dto, product fields from product, retailerKey from `adapter.retailerKey`, priceCents from `product.priceCents`, platformFeeCents from calculated fee.
- Note: Stripe PaymentIntent creation is NOT in this plan -- Plan 02 adds it. For now, set `stripePaymentIntentId: null`. The mock adapter path works without a real charge.
- Call `adapter.placeOrder(product, { name: dto.shippingName, address1: dto.shippingAddress1, address2: dto.shippingAddress2, city: dto.shippingCity, state: dto.shippingState, zip: dto.shippingZip }, order.id)`.
- On success: update order with `{ status: 'ordered', retailerOrderId: result.retailerOrderId, confirmationNumber: result.confirmationNumber, estimatedDeliveryDate: new Date(result.estimatedDeliveryDate) }`.
- If `dto.giftRecordId` provided: update the GiftRecord source to `'ordered'`: `prisma.giftRecord.update({ where: { id: dto.giftRecordId, userId: user.id }, data: { source: 'ordered' } })`. Wrap in try/catch — non-critical.
- Audit log via `orderAudit.record({ orderId: order.id, userId: user.id, action: 'place', details: { retailerOrderId: result.retailerOrderId, priceCents: product.priceCents } })`.
- On adapter failure (catch `RetailerOrderError`): update order status to `'failed'`, audit as `'place_failed'`, throw `InternalServerErrorException('Order placement failed. Please try again.')`.
- Return the full order record.

`async cancel(userId: string, orderId: string, reason?: string)`:
- Find order: `prisma.order.findFirst({ where: { id: orderId, userId } })`. Throw `NotFoundException('Order not found')` if missing.
- Validate cancel window: `if (!order.placedAt || (Date.now() - order.placedAt.getTime()) >= 2 * 60 * 60 * 1000) throw new BadRequestException('Cancel window has closed. Order cannot be cancelled.')`.
- Validate status is cancellable: `if (!['pending', 'ordered'].includes(order.status)) throw new BadRequestException('Order cannot be cancelled in its current state.')`.
- Call `adapter.cancelOrder(order.retailerOrderId || order.id)`. Catch errors and audit as `'cancel_failed'`, re-throw as BadRequestException.
- Note: Stripe refund is NOT in this plan -- Plan 02 adds it.
- Update order: `{ status: 'cancelled', cancelledAt: new Date(), cancelReason: reason || null }`.
- If order has `giftRecordId`: revert GiftRecord source to `'suggestion'`: `prisma.giftRecord.update({ where: { id: order.giftRecordId }, data: { source: 'suggestion' } })`. (Per D-11: "gift status reverts to approved (suggestion stays selected)").
- Audit log as `'cancel'`.
- Return updated order.

`async list(userId: string, query: ListOrdersDto)`:
- Page and limit with defaults: `page = query.page || 1, limit = query.limit || 20`.
- Build where clause: `{ userId }`. If `query.status`, add `status: query.status`.
- Count total, fetch with `skip: (page - 1) * limit, take: limit, orderBy: { createdAt: 'desc' }, include: { person: { select: { name: true } } }`.
- Return `{ data: orders, meta: { page, limit, total } }`.

`async getById(userId: string, orderId: string)`:
- `prisma.order.findFirst({ where: { id: orderId, userId }, include: { person: { select: { name: true } }, giftRecord: { select: { id: true, title: true, source: true } } } })`.
- Throw `NotFoundException('Order not found')` if missing.
- Compute `cancelWindowSecondsLeft`: if order.placedAt exists, `Math.max(0, Math.floor((order.placedAt.getTime() + 2 * 60 * 60 * 1000 - Date.now()) / 1000))`, else 0.
- Return `{ ...order, cancelWindowSecondsLeft }`.
